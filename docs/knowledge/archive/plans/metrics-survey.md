# Метрики качества извлечения сюжетных линий — обзор литературы

Март 2026. Обзор современных подходов к оценке качества narrative extraction / storyline clustering.

## Наши текущие метрики

| Метрика | Формула | Что измеряет |
|---------|---------|-------------|
| **Coverage** | доля событий с non-null storyline | Полнота разметки |
| **Consistency (ARI)** | Adjusted Rand Index по сетке (episode, character) → storyline | Стабильность между прогонами |
| **Score** | coverage × consistency | Композитный показатель |

## Что используют Balestri & Pescatore (ICAART 2025)

Статья: [Multi-Agent System for AI-Assisted Extraction of Narrative Arcs in TV Series](https://arxiv.org/html/2503.04817v1)

- Character entity accuracy: 98.4% (61/62)
- Anthology arc precision: 89.3% (25/28)
- Нет recall, F1 или систематических метрик для Soap/Genre-Specific арок
- Сравнение с одной экспертной разметкой (Grey's Anatomy S1)
- Авторы сами признают — это proof-of-concept, не строгая оценка

**Вывод**: планка невысокая, можно значительно улучшить.

## Метрики без разметки (annotation-free)

### Кластерные метрики (internal validation)

Поскольку извлечение линий — это по сути кластеризация (события → кластеры-линии), применимы стандартные метрики кластеризации.

| Метрика | Как работает | Диапазон | Что нужно |
|---------|-------------|----------|-----------|
| **Silhouette Coefficient** | Насколько событие похоже на свой кластер vs ближайший чужой | [-1, 1] | Эмбеддинги событий |
| **Davies-Bouldin Index** | Отношение внутрикластерного разброса к межкластерному | [0, ∞), меньше лучше | Эмбеддинги событий |
| **Calinski-Harabasz Index** | Отношение межкластерной дисперсии к внутрикластерной | больше лучше | Эмбеддинги событий |

**Как реализовать**: эмбедить текст каждого события (sentence-transformers или OpenAI embeddings), затем sklearn.metrics для расчёта.

### Нарративная когерентность

Из статьи [Narrative Trails (Text2Story 2025)](https://arxiv.org/html/2503.15681v1):

| Метрика                | Что измеряет                                                                                                             |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Minimum Coherence**  | Самое слабое звено в линии — минимальная похожесть между соседними событиями                                             |
| **Reliability**        | Геометрическое среднее когерентности по последовательным событиям в линии                                                |
| **Coherence function** | `θ(u,v) = sqrt(S(z_u, z_v) · T(ẑ_u, ẑ_v))` — комбинация угловой похожести эмбеддингов и топик-похожести (Jensen-Shannon) |
| **nDTW**               | Dynamic Time Warping между извлечённой и референсной линией, нормализованный по длине пути                               |
| **DTW Similarity**     | Средняя cosine similarity вдоль выравнивания DTW                                                                         |
|                        |                                                                                                                          |

**Применимость**: coherence и reliability можно считать без разметки. DTW-метрики нужны для сравнения с эталоном.

### LLM-as-a-Judge

Из статьи [LLM-as-a-Judge for Narrative Coherence (Electronics, 2025)](https://www.mdpi.com/2079-9292/14/13/2735):

- LLM-судья достигает **корреляции Пирсона ~0.65** с математическими метриками когерентности
- **Надёжность между прогонами ICC > 0.92** — очень стабильно
- Простые промпты работают почти так же хорошо, как сложные (90%+ от сложных подходов)
- Дёшево, не нужна разметка, даёт "человекоподобную" оценку

### Метрики из NLP-литературы по оценке нарративов

Из обзора [What Makes a Good Story and How Can We Measure It? (2024)](https://arxiv.org/html/2408.14622v1):

| Метрика | Что делает |
|---------|-----------|
| **CTRLEval** | Text infilling для оценки cohesion, consistency, relevance. Без эталона |
| **COHESENTIA** | Предложение за предложением оценивает локальную и глобальную связность |
| **BARTScore** | Вероятности токенов BART для оценки когерентности |
| **G-EVAL** | LLM-промпт с пошаговым рассуждением, гибкие измерения |
| **Perplexity** | Низкая perplexity конкатенированных событий линии = хорошая связность |
| **Distinct-n** | Доля уникальных n-грамм — детектор слишком похожих линий |

## Метрики с золотым стандартом (external validation)

Нужна экспертная разметка хотя бы для одного шоу (события → эталонные линии).

| Метрика | Что измеряет | Преимущество над ARI |
|---------|-------------|---------------------|
| **NMI (Normalized Mutual Information)** | Взаимная информация между предсказанием и эталоном, [0, 1] | Робастнее для кластеров разного размера |
| **AMI (Adjusted Mutual Information)** | NMI с поправкой на случайность | Строже NMI |
| **V-measure** | Гармоническое среднее homogeneity и completeness | Раскладывается: видно, проблема в пере-дроблении или недо-дроблении |
| **B-cubed F1** | Precision/recall на уровне каждого события | Стандарт в coreference resolution (похожая задача) |
| **F1 macro** | Средний F1 по линиям | Используется в SemEval-2025 Task 10 |
| **BERTScore F1** | Семантическая похожесть сгенерированных описаний с эталонными | Для оценки описаний линий (name, goal, etc.) |

## Смежные бенчмарки

| Название                               | Год  | Что оценивает                                                                                    |
| -------------------------------------- | ---- | ------------------------------------------------------------------------------------------------ |
| **SemEval-2025 Task 10**               | 2025 | Извлечение нарративов из новостей. F1 macro + BERTScore                                          |
| **SeriesBench (CVPR 2025)**            | 2025 | Понимание сериалов — 28 задач по 105 сериалам                                                    |
| **NarraBench**                         | 2025 | Таксономия 50 аспектов нарратива. Только 27% задач покрыто существующими бенчмарками             |
| **Topic Detection and Tracking (TDT)** | NIST | Классика: miss rate, false alarm rate. Концептуально близко (новости → топики ≈ события → линии) |

## Стабильность LLM между прогонами

Исследования 2024-2025 подтверждают:
- Разброс точности до **10% между идентичными прогонами** даже с temperature=0
- Агрегация по **3-5 прогонам** резко повышает стабильность
- Наш ARI-подход — валидный, литература поддерживает измерение inter-run agreement как первоклассную метрику

## Практические рекомендации

### Быстро добавить (без разметки, без больших затрат)

1. **Silhouette Score** — эмбеддинги событий + sklearn. Объективная метрика качества кластеризации.
2. **Within-storyline coherence** — средняя попарная похожесть событий одной линии.
3. **Storyline separation** — средняя между-линейная похожесть (ниже = лучше, линии разные).
4. **LLM-as-judge** — промпт для оценки каждой линии по рубрике.

### Средний горизонт (нужна одна экспертная разметка)

5. **V-measure** — вместо ARI, диагностирует тип ошибки.
6. **B-cubed F1** — стандарт для coreference-подобных задач.
7. **NMI** — робастнее ARI для неравных линий.

### Долгосрочно (инфраструктура)

8. **Narrative coherence из Narrative Trails** — нужны хорошие эмбеддинги + реализация.
9. **Multi-run aggregation** — прогон 3-5 раз, усреднение.
10. **SeriesBench-подобный бенчмарк** — золотая разметка для нескольких шоу разных типов.
