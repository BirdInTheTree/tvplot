# Synopsis Authoring Protocol

Протокол написания синопсисов эпизодов для пайплайна tvplotlines.

## Задача

Напиши prose outline эпизода — пересказ ключевых событий в хронологическом порядке. Результат должен быть достаточным для восстановления Story DNA каждой сюжетной линии (hero, goal, obstacle, stakes) и распределения событий по линиям.

## Правила

1. **Каждое предложение = beat** — событие, меняющее ситуацию. Если в предложении нет конфликта или изменения — оно не нужно.
2. **Действия и решения, не атмосфера.** Пиши что персонаж сделал и почему, а не как выглядела сцена.
3. **Персонажи по имени.** При первом появлении — полное имя + роль/отношение. Далее по имени.
4. **Каузальность явная.** Каждое событие должно следовать из предыдущего. Не «случайно», а «потому что».
5. **Все линии покрыты.** A, B, C и runners. Даже если B-story занимает одно предложение — она упомянута.
6. **Конфликт обязателен.** Если ты не можешь сформулировать конфликт в сцене — эта сцена не нужна в синопсисе.
7. **Минимум 3 нарративных нити на эпизод.** Кейс/A-story + минимум две personal/serialized арки. В процедуралах недостаточно описать только кейс — каждый эпизод содержит развитие персональных линий основного каста (конфликты в отношениях, карьерные решения, внутренние кризисы). Если в эпизоде нет явных B/C-линий — укажи это явно.
8. **Serialized контент ≥ 30% текста.** Синопсис, где 100% — кейс недели, бесполезен для извлечения сюжетных линий. Пайплайн не может извлечь то, чего нет в тексте.
9. **Фактическая точность обязательна.** Синопсис должен быть основан на реальном содержании эпизода, верифицированном по нескольким источникам. LLM-галлюцинации (выдуманные пациенты, несуществующие сцены) полностью обесценивают результат.

## Формат

- Сплошной текст, абзацы по сценам/локациям
- **300–500 слов** на эпизод (увеличено для покрытия serialized арок)
- **8–15 beats** на эпизод
- Язык оригинала шоу
- Без подзаголовков, без bullet points
- Без авторских оценок и отсылок к будущим эпизодам
- Хронологический порядок внутри эпизода; флэшбеки указывать явно

## Что включать

- Действия персонажей и их последствия
- Решения и мотивацию (если ясна из контекста)
- Конфликты и их исходы
- Ключевые реплики — только если они меняют ситуацию
- Появление новых персонажей с минимальным контекстом

## Что НЕ включать

- Визуальный стиль, музыку, режиссуру, монтаж
- Интерпретации, символизм, авторские оценки
- Слово "Meanwhile"
- Спойлеры из будущих эпизодов
- Пересказ событий прошлых эпизодов (только если персонаж в этом эпизоде о них узнаёт)

## Количество beats по типу шоу

| Тип | A-story | B-story | C / runner |
|---|---|---|---|
| Procedural | 5–8 | 2–4 | 1–2 |
| Serial | 4–6 | 3–5 | 1–3 |
| Ensemble | 2–4 per line | 2–4 per line | 1–2 per line |

## Авторство

Синопсисы написаны авторами проекта на основе общедоступного содержания эпизодов. Это derivative work — пересказ своими словами, не копия внешних источников.

Для статьи: "Episode synopses were authored by the researchers based on publicly available episode content, following a standardized protocol to ensure consistent granularity across different franchise types."
