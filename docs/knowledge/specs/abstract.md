# Abstract


tvplotlines, a Python library that answers these questions:
How do I figure out which is my A plot and which is my B plot?
How do you balance A, B, C stories without one getting lost?
How do you break down a show to understand its structure?
How do you know when a subplot is a B story vs just a scene?
How does [show] manage 5 storylines without losing the audience?
Pick a show you love and pull it apart to see how it works


TV episodes run two or more plotlines at once—an A story that takes most of the screen time, B and C stories that cut against it or echo it. Three per episode is standard; ensemble shows often have four or five. Picking which plotlines to tell, how fast to move them, when to collide them—that is the architecture of a show. Dialogue, direction, performance are built on top. If the architecture is weak, no amount of good design will fix it.

tvplotlines, a Python library that extracts this architecture from episode synopses using a multi-pass LLM   pipeline.
