# tvplotlines Standalone HTML App — Spec

## Суть

Один скачиваемый HTML-файл — полноценное приложение для анализа сюжетных линий сериалов. Открыл в браузере — работает. Ноль установки, данные не покидают компьютер.

## Целевая аудитория

Сценаристы 55+, минимальные технические навыки. Не пользуются терминалом. Боятся отдавать сценарии сторонним сервисам, но доверяют своему LLM (Claude/ChatGPT).

## Пользовательский flow

### Первый запуск

1. ПЕРВЫЙ РАЗ Открыл HTML → **Welcome-экран**: кнопка "Play me". В углу "Skip".
2. Нажал Play → **Онбординг-демо**: анимированная симуляция пайплайна на Breaking Bad. Проходы появляются один за другим, грид заполняется в реальном времени. Пользователь видит как работает инструмент.
3. В конце демо → **"Подключить свой LLM"**: выбор Claude / OpenAI, поле для API ключа, инструкция со скриншотами + ссылка на видео "Как получить АПИ Клод/ОпенИИ - чье-то найти". Кнопка "Пропустить" — остаться на демо-данных.
4. После подключения (или пропуска) → **Рабочий экран**: грид с данными Breaking Bad.

### Повторный визит

- API ключ уже в localStorage — не спрашивает
- Последние результаты сохранены — открывается на том же месте
- Меню: "Посмотреть онбординг", "Настройки LLM", "Загрузить синопсисы"

### Работа со своими данными

1. Drag & drop txt-файлов синопсисов (или кнопка "Выбрать файлы")
2. Прогресс: "Проход 1/5: извлекаю персонажей..."
3. Результат появляется в гриде
4. Результат автоматически сохраняется в localStorage

### Накопление

- Все обработанные сериалы сохраняются в localStorage
- Dropdown для переключения между сериалами
- Можно зайти через неделю — всё на месте

## Экраны

### Welcome

- Фон: японский минимализм
- Кнопка "Play me" — запускает онбординг
- "Skip →" в правильном месте для такой фукнции

### Онбординг-демо

Кодированная анимация на реальных экранах приложения (не видео):

1. Skip-экран с кнопкой → нажимается
2. Появляется окно "выбери LLM" → выбирается Claude (чередовать Claude/OpenAI рандомно?)
3. Вставляется ключ (анимация набора)
4. Появляется зона загрузки файлов → один файл через кнопку, второй drag & drop
5. Запускается пайплайн — прогресс проходов 1-5, грид заполняется в реальном времени
6. Результат: грид, аналитика, экспорт — показываем каждую фичу
7. Всё озвучено подписями/тултипами (не аудио)

### Подключение LLM

- Модальное окно поверх грида с демо-данными
- Две кнопки: Claude / OpenAI
- По выбору: поле для API ключа + пошаговая инструкция (3 шага со скриншотами)
- Ссылка на видео "как получить ключ"
- Ключ сохраняется в localStorage
- "Пропустить" — закрывает окно, работаешь с демо-данными

### Рабочий экран — Грид

Из tvplotlines-app:
- Toolbar: название сериала, dropdown выбора сериала, кнопки экспорта, dark mode toggle
- Фильтры: сезон, персонажи, функции событий
- Таблица: сюжетные линии (ряды) x эпизоды (колонки)
- Первая колонка sticky: имя линии, ранг (A/B/C), персонаж-драйвер
- Ячейки: карточки событий, цвет по функции (setup серый, climax красный, resolution зелёный...)
- Клик по карточке: модальное окно с деталями события (read-only в v1)

### Рабочий экран — Аналитика

Из tvplotlines-app:
- Season scorecard: таблица с span-bar
- Arc map: тепловая карта напряжения
- Episode pulse: stacked bars
- Convergence moments: таймлайн взаимодействий линий
- Character weight: горизонтальные бары

## Экспорт

Четыре формата, кнопки в toolbar:

1. **JSON** — наш формат tvplotlines, тот же что генерирует библиотека. Для импорта обратно.
2. **XLS** — таблица: строки = события, колонки = эпизод, сюжетная линия, функция, персонажи, текст. Библиотека SheetJS (inline).
3. **TXT** — plain text, структурированный по эпизодам и линиям.
4. **Final Draft (.fdx)** — XML-формат Final Draft. Outline с beat board: каждая сюжетная линия = storyline, каждое событие = beat с функцией и описанием.

## Техническая архитектура

### Один HTML-файл содержит:

- `<style>` — минимализм, почти олдскульный дизайн первого интернета, но чисто без нарочитой аналоговости, темная тема обязательно должна быть, проверить чтобы ВСЕ элементы ОДИНАКОВО читались и работали при переключении темы с темной на светлую и наоборот
- `<script>` — весь JS:
  - Онбординг-анимация (таймеры, поэтапное появление)
  - LLM-пайплайн (5 проходов, промпты из tvplotlines, fetch + SSE streaming)
  - Грид-рендеринг
  - Аналитика-рендеринг
  - Экспорт (SheetJS inline для XLS, XML-генерация для .fdx)
  - localStorage менеджер (ключ, результаты, настройки)
  - Router (Welcome / Grid / Analytics — переключение через JS, без перезагрузки)
- `<script type="application/json" id="demo-data">` — Breaking Bad S01 результат
- HTML-разметка всех экранов

### LLM-пайплайн в браузере

Порт 5 проходов tvplotlines из Python в JS:
- Промпты — те же самые (копируются из библиотеки)
- Формат JSON — тот же
- API вызов — fetch напрямую к api.anthropic.com или api.openai.com
- Стриминг — SSE для прогресса
- CORS: Anthropic API поддерживает browser requests с API ключом в заголовке

### Хранение данных

localStorage:
- `tvplotlines_api_key` — зашифрованный API ключ
- `tvplotlines_provider` — "anthropic" | "openai"
- `tvplotlines_results` — объект {series_name: result_json}
- `tvplotlines_settings` — dark mode, последний открытый сериал
- `tvplotlines_onboarding_seen` — boolean

### Зависимости (всё inline)

- SheetJS (для XLS экспорта) — ~200KB минифицированный
- Никаких CDN, никаких внешних запросов кроме LLM API

## Связь с библиотекой tvplotlines

`tvplotlines run my-series/ --html` генерирует этот же HTML с данными пользователя вместо Breaking Bad. Пайплайн уже выполнен на стороне Python, HTML используется только для просмотра и экспорта.

## Размер файла

~1-2 MB (SheetJS ~200KB + CSS ~15KB + JS ~100KB + демо-данные ~50KB + онбординг-ассеты).

## Чего НЕ делаем в v1

- Редактирование (merge/split/правка событий)
- Платный аккаунт (прокси-бэкенд)
- Screen assistant как онбординг-помощник
- Мультиязычность интерфейса
